iOS Localization problem with XIB, Localizable.strings

앱 지역화 작업 중

xib 파일에 지역화를 적용한 뒤 빌드를 해보니, 해당 파일을 로드 하는 시점에서 크래시가 발생한다.

*** Terminating app due to uncaught exception 'NSInternalInconsistencyException', reason: 'Could not load NIB in bundle: 'NSBundle < /var/mobile/Applications/_____/_____.app> (loaded)' with name '__MY NIB STRING HERE___''

위와 같은 크래시 로그가 남았는데, 이는 xib파일이 외부에서 변경되었을 때 xcode 프로젝트 파일에 변경사항이 반영이 되지 않아 생기는 문제 였다. 간단히 해당파일의 레퍼런스를 삭제하고 다시 프로젝트로 추가하는 것으로 해결했다.

그리고 다음으로 Localizable.strings 파일이 적용이 안되는 문제가 생겼다.

첫번째 원인은 잘못 된 문자열 형식이 들어 간것이었다.

[code language="objc"]
"ABC" = "가나다"; // 바른 형식
@"ABC" = "가나다"; // 틀린 형식
[/code]

해당 문자열을 복사해서 붙여넣는 과정에서 골뱅이(@) 가 들어간 것이었다. 빌드 시 크래시가 나는 경우도 있고, 나지 않는 경우도 있다. 나의 경우엔 처음에 크래시가 발생하여 문제를 해결한 뒤, 재현을 위해 코드를 고쳐보았는데 크래시가 나지 않았다.

두번째 원인은 동일한 키가 중복 된 것.

[code language="objc"]
"ABC" = "가나다";
"DEF" = "라마바";
"ABC" = "가나다";
[/code]

이 경우 빌드를 하는데 아무런 문제가 없기 때문에 이와 같은 문제가 발생했을 때 가장 먼저 중복 체크를 해보는게 좋겠다.
